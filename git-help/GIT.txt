1. Завести аккаунт на git hub

2. Установить GIT BASH + GIT GUI

3. Стащить репозиторий [git clone https://github.com/%user_login%/%repo_name%.git],
либо выбрать локальную папку и создаём репозиторий локально
(можно с помощью git gui > "Create New Repository" - если не был создан ранее)
либо путь папки + git init

4. Вызываем Git Bash в нашей папке проекта и настраиваем:
git config user.name ivan.ivanov
git config user.email ivanov@example.com

5. Получаем изменения с репозитория, если были
git pull

6. Проверяем наличие назакоммиченых файлов 
git status

7. Указываем какие файлы нужно коммитить:
один файл(указываем путь конкретного файла)
git add %file_path%
или все в коммит
git add .

8. Делаем собственно коммит (откроет окно для добавления комментария)
git commit
или добавить коммент сразу
git commit -m "%commit_message%"

9. Проверяем истроию изменений командой
git log

-----
10. Указываем дирректорию Git Hub
git remote add origin https://github.com/mashuxa/storage.git

10. Запуливаем на удаленный репозиторий
git push -u origin master

Если возникает ошибка, возможно не слиты последние изменения
 нужно сделать git pull


**********************************
Для решения проблемы с crlf:
git config core.autocrlf false 



*******************
git remote -v       проверить сокращенные именна для удаленных репозиториев (origin и другие)
